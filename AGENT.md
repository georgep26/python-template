### When Building Applications on Top of This Template

When designing and implementing applications using this template as a foundation:

1. **Preserve the Directory Structure**
   - Maintain the standard directory layout (`config/`, `src/`, `tests/`, `infra/`, `scripts/`, `docs/`, `notebooks/`)
   - Follow the organizational patterns established in the template

2. **Replace Example Code with Real Application Logic**
   - The template contains **example/placeholder code** (e.g., `src/main.py`, `tests/test_main.py`)
   - **Swap out these examples** with actual application-specific code
   - Keep the same file structure and naming conventions, but implement real functionality

3. **Update Configuration Files**
   - Modify `config/app_config.yml` with actual application settings
   - Update `environment.yml` with project-specific dependencies
   - Adjust `pyproject.toml` with relevant project metadata and tool configurations

4. **Customize Documentation**
   - Update `README.md` to reflect the actual application purpose and features
   - Modify or extend documentation in `docs/` as needed
   - Update the directory structure diagram in `README.md` if you add new directories

5. **Maintain Template Standards**
   - Follow the coding standards and practices outlined in the template
   - Use the pre-commit hooks and code quality tools configured in the template
   - Adhere to the changelog format and versioning standards
